Nome do Projeto: Lista de Tarefas CLI
Descrição:
Desenvolveremos um aplicativo de linha de comando (CLI) que permitirá aos usuários gerenciar uma lista de tarefas. Os usuários poderão adicionar, 
visualizar, marcar como concluídas e remover tarefas da lista.

Funcionalidades Principais:
Adição de Tarefas:

Implemente uma opção para permitir que os usuários adicionem novas tarefas à lista.
Utilize a biblioteca readline-sync para interagir com o usuário e receber a entrada da nova tarefa.

Visualização da Lista de Tarefas:

Desenvolva uma função para exibir todas as tarefas atualmente na lista.
Exiba as tarefas de forma numerada para facilitar a identificação.
Marcação de Tarefas Concluídas:

Permita que os usuários marquem uma tarefa como concluída.
Solicite ao usuário o número da tarefa que deseja marcar como concluída e atualize o status da tarefa correspondente.
Remoção de Tarefas:

Implemente uma opção para permitir que os usuários removam tarefas da lista.
Solicite ao usuário o número da tarefa que deseja remover e exclua-a da lista.
Passos de Implementação:
Definição da Estrutura de Dados:

Escolha uma estrutura de dados adequada para armazenar as tarefas da lista, como um array de objetos em JavaScript.
Adição de Tarefas:

Crie uma função para adicionar novas tarefas à lista.
Solicite ao usuário o nome da nova tarefa usando a biblioteca readline-sync.
Visualização da Lista de Tarefas:

Desenvolva uma função para exibir todas as tarefas atualmente na lista.
Percorra a lista de tarefas e exiba cada uma delas no terminal, numerando-as para facilitar a identificação.
Marcação de Tarefas Concluídas:

Implemente uma opção para marcar uma tarefa como concluída.
Solicite ao usuário o número da tarefa que deseja marcar como concluída e atualize o status correspondente na estrutura de dados.
Remoção de Tarefas:

Desenvolva uma função para remover tarefas da lista.
Solicite ao usuário o número da tarefa que deseja remover e exclua-a da lista.
Testes e Validação:

Teste o aplicativo com diferentes cenários, adicionando, visualizando, marcando como concluídas e removendo tarefas.
Valide a entrada do usuário e trate casos de uso inválidos ou excepcionais adequadamente.
Documentação e Entrega:

Documente o projeto com comentários claros em seu código, explicando sua estrutura e funcionamento.
Prepare uma breve documentação ou instruções de uso para o usuário.
Empacote o projeto em um formato adequado para entrega, como um arquivo ZIP contendo todos os arquivos necessários.
